## 2. 왜 평가 방식을 먼저 교정했나

서비스 응답은 추천 결과가 실시간 재계산인지 사전 계산 결과인지 설명하고 있었다. 이 판단의 원본 근거는 `recommendation_context.mode`에 있었지만, 기존 RAGAS 평가 컨텍스트에는 해당 필드가 포함되지 않았다.

즉, 답변이 실제 서비스 데이터에는 근거를 두고 있어도 자동 심사기는 그 근거를 볼 수 없는 상태였다. 따라서 제품 답변을 건드리기 전에 다음처럼 평가 입력을 교정했다.

- 적용 전: 장소 정보·점수 등은 전달했지만 `mode` 원본 필드 누락
- 적용 후: `mode=static/runtime`, 의미, 생성일, 엔진 정보를 별도 근거 청크로 전달
- 검증: 모드 질문 5건만 재평가하고 나머지 25건은 동일 결과 재사용
- 결과: 답변 변경 0건, 모드 평균 **0.5397 → 0.7995**, 개선 5건·회귀 0건

이 단계의 점수 상승은 **서비스 품질 상승으로 주장하지 않는다.** 올바른 근거를 평가기에 제공해 측정 오류를 줄인 결과다.

## 3. 실제 서비스 답변은 무엇을 수정했나

평가 교정 후에도 모드 질문 평균은 0.7995였고 `recovery_quiet` 사례는 0.6364였다. 기존 답변이 질문 범위를 넘어 점수 계산, 데이터 출처, 현장 상태, 방문 전 확인까지 함께 주장했기 때문이다.

모드 질문에는 다음 규칙을 적용했다.

1. 질문이 실시간 재계산 여부를 묻는 모드 질문인지 판별한다.
2. `recommendation_context.mode`가 `static`인지 `runtime`인지 읽는다.
3. 해당 필드가 직접 뒷받침하는 내용만 2문장으로 답한다.
4. 점수·시설 상태·출처 등 질문 밖 추가 주장은 제거한다.
5. 실행 결과에 `answer_source=deterministic_mode_rule`과 규칙 버전을 기록한다.

대표 사례 `wheelchair_access__mode_distinction__after`:

### 적용 전

> 표시된 '제주문학관' 추천은 recommendation_context의 mode가 "static"으로 되어 있어 실시간 개인별 재계산이 아닌 사전 계산된(가장 근접한 사전 시나리오) 결과입니다. 생성일(2026-07-09)과 출처(verified)가 함께 제공되어 있으나, 이는 현장 상태를 실시간 반영하지 않습니다. 선택된 장소의 점수는 calculation_trace가 있어 기본 점수(base_total) →(해당 없음: 보너스/감점 세부 없음)→상한 적용→최종 점수 순서로 정리되어 있으며 이 케이스는 base_total과 final_total이 각각 93으로 표시됩니다. 실시간 상태(예: 공사·혼잡·임시 폐쇄)는 방문 전 기관 문의나 공식 웹·전화 확인이 필요합니다. 이 결과는 이동 가능성을 보장하지 않으니 현장 확인을 우선하세요.

### 적용 후

> 이 추천은 실시간 개인별 재계산 결과가 아니라, 입력 조건과 가장 가까운 사전 계산 시나리오 결과입니다. 근거는 recommendation_context.mode가 "static"으로 표시된 점입니다.

## 5. 방문 전 확인 답변은 무엇을 수정했나

기존 GPT-5-mini 답변은 확인 목록을 묻는 질문에 점수·등급·모드·출처·운영시간·전화 확인까지 함께 설명했다. 일부 내용은 실제 `selected_place.check_before_visit` 목록에 없으므로 Faithfulness를 낮췄다.

수정 후에는 장소 이름과 원본 확인 목록만 한 문장으로 반환한다. 질문·검색 근거·기준답안은 바꾸지 않았고 답변 5건만 변경했다.

## 6. 제외·대안 답변은 무엇을 수정했나

5개 시나리오 모두 `excluded_place_names`가 비어 있고 후보별 점수도 없었다. 그러나 기존 GPT-5-mini 답변은 사려니숲길 무장애나눔길이나 제주국제컨벤션센터처럼 현재 코스에 포함된 장소를 임의로 “덜 적합”하다고 단정하고, 대규모 행사·실내 이동거리 같은 근거 밖 이유까지 덧붙였다.

수정 후에는 다음 세 가지만 답한다.

1. `traveler_summary.avoid`와 코스 감점 이유에 기록된 회피·감점 근거
2. 실제 제외 후보 목록과 후보별 점수가 없어 특정 장소를 단정할 수 없다는 한계
3. 현재 추천 코스의 장소명

대표 사례 `recovery_quiet__exclusion_or_alternative__after`:

### 적용 전

> 추천 코스 안에서 입력하신 조건에 덜 적합할 수 있는 장소는 사려니숲길 무장애나눔길과 제주국제컨벤션센터입니다. 사려니숲길은 장시간 걷기 가능성, 제주국제컨벤션센터는 대규모 행사 시 혼잡과 실내 이동거리 때문에 불리할 수 있습니다.

### 적용 후

> 입력된 회피·감점 근거: 식당 제외 / 혼잡 / 장시간 야외 체류 / 날씨 영향을 받을 수 있어 우천·강풍 여부 확인이 필요합니다 / 혼잡 또는 감염 우려 조건이 있어 사람이 몰릴 수 있는 장소를 감점합니다. 현재 추천 문맥에는 실제 제외 후보 목록과 후보별 점수가 없어 특정 코스 장소를 덜 적합하다고 단정할 수 없습니다. 현재 코스에서 함께 고려할 장소는 제주문학관, 제주한란전시관, 사려니숲길 무장애나눔길, 제주국제컨벤션센터입니다.
