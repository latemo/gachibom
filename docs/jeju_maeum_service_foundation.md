# 제주의마음 여행 AI 서비스 기준서

## 1. 서비스 원칙

제주의마음 여행 AI는 제주 관광약자를 위한 무장애 여행 판단 보조 서비스다.

이 서비스의 핵심은 "추천"이 아니라 "사용자가 자기 상황에 맞게 판단할 수 있도록 돕는 것"이다.

처음부터 고정할 원칙:

- 공식 또는 검수된 접근성 정보에 근거한다.
- 정보가 부족하면 부족하다고 표시한다.
- 여행 가능성을 보장하지 않는다.
- 의료 판단을 하지 않는다.
- 사용자의 이동 부담을 과소평가하지 않는다.
- 결과에는 항상 출처, 확인 필요 항목, 안전 문구가 포함된다.
- 운영자가 데이터를 수정하고 갱신할 수 있는 구조를 유지한다.

## 2. 한 문장 정의

관광약자의 이동 조건과 제주 접근성 정보를 대조해, 무리 없는 여행 코스와 방문 전 확인사항을 제안하는 AI 여행 지원 서비스.

## 3. 핵심 사용자군

1차 서비스는 아래 사용자군을 모두 고려한다.

- 휠체어 이용자
- 고령자
- 암환자 또는 치료 후 회복 중인 여행자
- 임산부
- 유모차 동반 가족
- 일시적으로 몸이 불편한 관광객
- 보호자, 가족, 복지기관 담당자

우선 완성도를 높일 대표 사용자군:

- 보호자와 함께 여행하는 고령자 또는 회복 중인 여행자
- 휠체어 이용자
- 유모차 동반 가족

## 4. 서비스가 반드시 해결해야 할 문제

사용자는 보통 다음 질문에 답을 얻고 싶어 한다.

- 내가 이 장소에 갈 수 있을까?
- 얼마나 걸어야 할까?
- 계단이나 경사가 있을까?
- 장애인 화장실이 있을까?
- 주차장에서 입구까지 멀까?
- 중간에 쉴 수 있을까?
- 사람이 너무 많거나 오래 서 있어야 할까?
- 현장에서 무엇을 미리 확인해야 할까?

서비스는 이 질문에 대해 단정하지 않고, 근거와 확인사항을 함께 제공해야 한다.

## 5. 서비스가 하지 않는 것

아래 기능은 1차 서비스 범위에서 제외한다.

- 의료 상담
- 진단, 치료, 회복 가능성 판단
- 여행 가능성 보장
- 접근성 100% 보장
- 실시간 현장 상태 보장
- 예약, 결제, 보험, 병원 연계
- 개인 건강정보 장기 저장
- 장애 등급 또는 건강 상태 자동 판정

## 6. 정보 신뢰도 원칙

장소 정보는 신뢰도 등급을 가진다.

| 상태 | 의미 | 서비스 표시 |
| --- | --- | --- |
| verified | 공식 또는 운영자 검수 정보가 충분함 | 확인됨 |
| partial | 일부 정보만 확인됨 | 일부 확인 |
| needs_check | 중요 정보가 부족하거나 오래됨 | 확인 필요 |
| unavailable | 접근성 판단에 필요한 정보가 없음 | 정보 부족 |

정보가 부족한 장소는 높은 점수를 받을 수 없다.

## 7. 결과 표시 원칙

결과 화면에는 항상 다음 요소가 있어야 한다.

- 코스 또는 장소명
- 총점과 등급
- 적합 이유
- 감점 이유
- 확인 필요 항목
- 이동 부담 요약
- 편의시설 요약
- 출처
- 정보 확인일
- 안전 안내 문구

색상은 보조 수단일 뿐이다. 점수와 등급은 반드시 텍스트로도 표시한다.

## 8. 개인정보 원칙

1차 서비스는 민감한 건강정보 저장을 전제로 하지 않는다.

처리 원칙:

- 사용자의 자유 입력은 결과 생성에만 사용한다.
- 이름, 연락처, 주민번호, 병원명, 상세 진단명 입력을 요구하지 않는다.
- 건강 상태는 "이동 조건"과 "피해야 할 요소" 수준으로만 해석한다.
- 로그를 남길 경우 원문 저장을 피하고 비식별 요약만 저장한다.
- 사용자 피드백 수집 시 동의 문구를 분리한다.

## 9. 운영 원칙

서비스화하려면 운영자가 데이터를 계속 관리할 수 있어야 한다.

운영자가 할 수 있어야 하는 일:

- 장소 카드 추가
- 장소 카드 수정
- 출처 URL 업데이트
- 정보 확인일 갱신
- 확인 필요 항목 표시
- 위험 표현 차단 목록 수정
- 사용자 피드백 확인
- 부정확한 추천 사례 기록

초기에는 파일 기반 운영도 가능하지만, 데이터 구조는 나중에 관리자 페이지와 데이터베이스로 옮길 수 있어야 한다.

## 10. 출시 기준

출시 전 반드시 만족해야 하는 기준:

- 모든 결과에 출처가 표시된다.
- 모든 결과에 안전 안내 문구가 표시된다.
- 공식 근거가 부족한 장소는 "확인 필요"로 표시된다.
- 금지 표현이 결과에 나오지 않는다.
- 이동 부담이 큰 장소는 사용자 조건에 따라 감점된다.
- 사용자 조건 요약을 사용자가 확인할 수 있다.
- 결과가 추천 이유와 감점 이유를 함께 보여준다.

## 11. 성공 기준

초기 성공 기준은 사용량이 아니라 신뢰도다.

정량 기준:

- 출처 표시율: 100%
- 안전 문구 포함률: 100%
- 금지 표현 차단률: 100%
- 확인 필요 항목 표시율: 90% 이상
- 사용자 조건 반영률: 90% 이상

정성 기준:

- 보호자가 방문 전 무엇을 확인해야 하는지 이해한다.
- 사용자가 과장된 추천이라고 느끼지 않는다.
- 정보가 부족한 부분이 숨겨지지 않는다.
- 실제 운영자가 장소 정보를 갱신할 수 있다.
